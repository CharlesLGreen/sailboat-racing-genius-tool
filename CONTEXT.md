## Snipe app — critical conventions

- **Single-file architecture**: `src/server.js` (~11,000 lines) holds Express routes, SQL schema migrations, page templates, and inline client JS. No build step, no client bundle, no framework. Edit with surgical care.
- **`/forecast` page has 4 mutually-isolated `<script>` IIFEs**: original wind/tide/direction charts, Sailing Area Current Map (Leaflet), 7-Day NWS forecast, HRRR wind map, RainViewer radar map. **Do NOT introduce cross-IIFE coupling** — it has caused multi-day breakage. New sections go in their own `<div>` + new `<script>` IIFE.
- **Cross-IIFE communication uses one window event**: `forecast-location-changed` with `{lat, lon, name}` in `e.detail`. The Original forecast IIFE dispatches it; HRRR and Sailing Area Current Map listen. Also caches latest at `window.__forecastLocation`.
- **Inline `/manifest.json`, `/sw.js`, and `/icons/icon-{192,512}{x{192,512}}.png` routes are declared BEFORE `app.use(express.static(...))`** in the middleware chain. This is intentional — the static folder shadowed them historically and broke PWA validation. Don't move them.
- **PWA icons are generated by `sharp` from inline SVG**, not served from disk. The on-disk `src/public/icon-192.png` is dead code. Same for `src/public/manifest.json` and `src/public/sw.js`.
- **The `/hero8.jpg` route serves a base64-inlined image buffer at the bottom of `server.js`**, not a real disk file. Don't search for the file — it doesn't exist.
- **DB migrations** happen at startup with `try { db.exec("ALTER TABLE ...") } catch {}`. Always wrap new ALTERs the same way.
- **`renderPage(html, user, lang, isHome)`** wraps page content in the global header/nav/footer. Most page handlers call it. Login/register/landing pages skip it.
- **Race feed at `/`**: pulls 50 most recent logs from `users` who set `data_sharing = 'share'`, joined to `race_logs`, rendered by `homePage()`.

## Crew College — critical conventions

- **AGENTS.md** (`/AGENTS.md`) reminds you: "This is NOT the Next.js you know. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code." Next.js 16 + React 19 has breaking changes vs. earlier versions.
- **Image src URLs are absolute** (`https://my-project-tan-seven-56.vercel.app/images/...`), not relative. The domain is allowlisted in `next.config.ts` under `images.remotePatterns`.
- **Trustee page at `/trustee`** is gated by a session-based password set during initial setup. The page (`src/app/trustee/page.tsx`, 1705 lines) consumes the API at `/api/trustee` (`src/app/api/trustee/route.ts`, 658 lines). The API returns `{ assets, trustees, log }` from Neon's `trustee_assets`, `trustees`, `succession_log` tables.
- **`seedAssets()` writes the asset list once** during initial setup. Editing the seed array does NOT update existing rows in Neon. Manual UPDATE statements or a re-seed are required for content changes to appear on the live trustee page. Use the **Claude Context tab → Re-seed asset list from source** button to push the latest seed array to Neon.
